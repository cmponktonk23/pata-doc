# How To Change Grid (Floor) Size

In this tutorial, I will show you how to change the grid (floor) size in Room Building 
Starter Kit. `The build system in this asset is made base on grid system. And a grid 
is equal to a floor in this asset which means the floors in each office constitute the 
entire grid system.`


##  Limitations

Please let me list the limitations of this feature in advance:
1. The floor can only be square;
2. The wall length must be same with the floor size, which means a floor can only 
have four walls around it in four directions (up, down, left, right);
3. The floor size can‚Äôt be changed in play mode;
4. Consider different users have different requirements, if I add too much functions, 
some user may need to subtract them. So, here I just supply a place to change the 
grid size used by the build system scripts. User need to make sure the models used 
by the build system have the right size compatible with the floor;
5. When you design your build system models, their origins must at same location 
with the relative model‚Äôs origin in this asset. After you put your models into Unity 
as a prefab, please make sure its collider shape fits the model size like the built-in 
models in this asset.

##  Change grid size

By changing the grid size, the work has two parts: `Change the grid size value used by scripts` 
and `scale or replace the models used by build system`. The models used 
by build system are associated with the build logic and that‚Äôs why it can work 
precisely in this asset. So, we have some rules need to be followed during design your 
models. Here, we recommend that determine your floor size first before design all the 
other models. In this tutorial, all the model size unit is meter.

###  Change grid size used by scripts

Let‚Äôs define the floor size as 3.5m x 3.5m. The floor size used by scripts is defined 
under:
`‚ÄúAssets/RoomBuildingStarterKit/Assets/ScriptableObjects/BuildSystemSettings.asset‚Äù`

In its inspector window, you can see the default grid size is `2m`. In this asset, all the built-in 
models used by the build system are designed base on the grid size. Here, we change the 
value to `3.5m`.
After change this value, we have to change all these models‚Äôscales or replace to new models. 
I think most of the users will not use the low poly models inside this asset üòâ
